FinWorld Architecture

Seven-layer architecture for comprehensive financial AI development

Architecture Overview

FinWorld employs a layered, object-oriented architecture with seven core layers, each handling specific aspects of the financial AI workflow.

Figure 1: FinWorld's comprehensive seven-layer architecture

1

Configuration Layer

Built on mmengine for unified experiment management with registry mechanism and configuration inheritance.

Key Components:

  • Registry System: Flexible component management and instantiation
  • Configuration Inheritance: Support for configuration overrides and extensions
  • Experiment Management: Centralized configuration for reproducibility
  • Parameter Validation: Automatic configuration validation and error checking
2

Dataset Layer

Multi-source data acquisition, feature engineering, task-specific organization, and RL environment encapsulation.

Core Modules:

  • Downloader Module: Multi-source data acquisition (FMP, Alpaca, AKShare, TuShare)
  • Processor Module: Feature engineering and preprocessing (Alpha158 factors)
  • Dataset Module: Task-specific data organization
  • Environment Module: RL environment encapsulation
3

Model Layer

ML models, DL architectures, RL networks, and unified LLM interface with financial constraints.

Model Categories:

  • ML Models: Traditional ML algorithms (LightGBM, XGBoost)
  • DL Models: Neural architectures (Transformer, LSTM, VAE)
  • RL Models: Actor-critic networks with financial constraints
  • LLM Models: Unified interface for commercial and open-source LLMs
4

Training Layer

Optimizers, loss functions, schedulers, metrics, and task-specific trainers with distributed support.

Training Components:

  • Optimizers: Adam, AdamW, SGD with gradient centralization
  • Loss Functions: Regression, classification, and RL surrogate losses
  • Schedulers: Cosine, linear, and adaptive learning rate scheduling
  • Trainers: Task-specific training pipelines with distributed support
5

Evaluation Layer

Financial-specific metrics, visualization tools, and standardized evaluation protocols.

Evaluation Components:

  • Financial Metrics: ARR, SR, MDD, CR, SoR, VOL
  • Visualization: K-line charts, cumulative returns, compass plots
  • Standardized Protocols: Consistent evaluation across all tasks
  • Performance Tracking: Real-time monitoring and analysis
6

Task Layer

Time series forecasting, algorithmic trading, portfolio management, and LLM applications.

Supported Tasks:

  • Time Series Forecasting: Multi-step prediction with various horizons
  • Algorithmic Trading: Single-asset trading with risk management
  • Portfolio Management: Multi-asset allocation with constraints
  • LLM Applications: Financial reasoning and agent training
7

Presentation Layer

Auto-reporting, multi-channel publishing, and version control for systematic archiving.

Presentation Features:

  • Auto-reporting: LaTeX technical reports and HTML dashboards
  • Multi-channel Publishing: GitHub, GitHub Pages, and experiment tracking
  • Version Control: Systematic archiving and knowledge transfer
  • Visualization: Interactive charts and performance dashboards
